# 全局硬性禁令

> ⚠️ 违反任意一条 → **立即停止本轮**，记录原因到执行报告，不论是否达到数量上限。

## 1. 命令串行执行 & 强制随机间隔（最高优先级）

### 🔴 绝对禁令

```
❌ 严禁并发执行任何 CLI 命令
❌ 严禁使用固定间隔（必须每次重新随机）
❌ 严禁跳过 sleep（即使"感觉"不需要）
❌ 严禁在 sleep 期间执行其他命令
```

### ✅ 强制要求

```
✅ 所有命令必须逐条串行执行
✅ 写操作（post/like/delete-comment）：40-55 秒随机间隔
✅ 读操作（get/search/replies/my/download）：30-50 秒随机间隔
✅ 每次等待前必须重新生成随机数
✅ 宁可慢不可快 — 如果拿不准，取更长的间隔
```

### 📋 执行模板（必须使用）

```bash
# ═══ 单条命令执行模板 ═══
execute_with_delay() {
  local cmd="$1"
  local type="${2:-write}"  # write 或 read
  local delay
  
  if [ "$type" = "read" ]; then
    delay=$((30 + RANDOM % 21))  # 30-50 秒随机值
  else
    delay=$((40 + RANDOM % 16))  # 40-55 秒随机值
  fi
  
  echo "▶ 执行: $cmd"
  eval "$cmd"
  
  if [ $? -eq 0 ]; then
    echo "✓ 成功，等待 ${delay} 秒..."
    sleep "$delay"
  else
    echo "✗ 失败，跳过等待"
  fi
}

# ═══ 批量执行模板 ═══
commands=(
  'node cli.js search "AI Agent" --count 20'
  'node cli.js get <aweme_id> --all --depth 1'
  'node cli.js post <aweme_id> "评论内容"'
)

for cmd in "${commands[@]}"; do
  execute_with_delay "$cmd"
done
```

### 💡 原因

抖音对高频操作敏感，并发或过快间隔会触发风控（限流 / 验证码 / 封号）。这是**硬性下限**，宁可慢不可快。

### ⚙️ 执行细节

- 带 sleep 的 Bash 调用必须设置 `timeout: 120000`（2 分钟），避免提前杀掉。
- 单轮连续操作 ≤ 10 次；之后暂停 3–5 分钟再开下一轮。
- 同一账号每日 ≤ 3 轮（合计 ≤ 30 条写操作）。
- ⚠️ **超出 2 轮后风险显著上升**：触发 status_code=8/2053 / 验证码 / 限发概率显著高于 2 轮内。第 3 轮发现任何风控 → 立即停止，不再尝试本日写操作。

## 2. 不重复回复

### 🔴 绝对禁令

```
❌ 同一条评论 cid 一生只能被回复一次（自有视频 + 他人视频均如此）
❌ 同一作者短期内不重复（推广引流场景）：≥ 7 天才能再次评论同一作者
```

### ✅ 强制要求

```
✅ 执行前必须先调 node cli.js log --tail 200，扫所有 post 操作中带 --reply-to <cid> 的记录
✅ 把这些 cid 加入"已回复集合" REPLIED_CIDS，循环里逐条 if cid in REPLIED_CIDS: skip
✅ 这是累计禁令，不是单轮去重 — 跨日 / 跨轮均生效
```

### 📋 执行流程

```bash
# ═══ Step 1: 初始化 REPLIED_CIDS ═══
# 获取所有已回复的 cid
node cli.js log --tail 200 | grep -o '"reply_to":"[^"]*"' | cut -d'"' -f4 > /tmp/replied_cids.txt

# ═══ Step 2: 检查是否已回复 ═══
is_replied() {
  local cid="$1"
  grep -q "$cid" /tmp/replied_cids.txt 2>/dev/null
  return $?
}

# ═══ Step 3: 回复前检查 ═══
if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
else
  execute_with_delay 'node cli.js post <aweme_id> "内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
fi
```

## 7. 评论筛选过滤规则

> 💡 评论区数量庞大但有价值的少，盲目回复浪费时间且效果差。

### 核心原则

只回复**有真实互动价值**的评论，跳过无实质内容的。

### 跳过类型（遇到直接跳过）

| 类型 | 典型示例 | 跳过原因 |
|------|----------|----------|
| 纯表情/贴纸 | "🔥"、"❤️"、"👍" | 表情回复路线，不走文字 |
| 简短感叹 | "666"、"哈哈"、"..." | 无讨论空间 |
| 无意义水评 | "来了"、"前排" | 无实质内容 |

### 优先回复类型

| 类型 | 典型示例 | 回复策略 |
|------|----------|----------|
| 提问求助 | "怎么做到的？"、"求教程" | 提供有价值的回答 |
| 高质量反馈 | >15 字，提到视频具体细节 | 参与讨论，展示专业度 |
| 负面情绪 | 抱怨/失望/愤怒 | 先感谢再解释，不争辩 |

### 判断标准

```
✓ 评论有实质内容（不是纯感叹/水评）
✓ 回复后对方可能会继续讨论
✓ 话题与自身定位相关（AI/技术/编程）
✓ 回复能体现专业度或提供价值
```

## 8. 获取评论限制

### 🔴 绝对禁令

```
❌ 严禁一次性获取大量评论（容易触发风控）
```

### ✅ 强制要求

```
✅ 优先使用 --new 增量获取（只拉新评论，请求量最少）
✅ 首次获取可用 --all --depth 1（全量 + 嵌套回复）
✅ 后续获取必须使用 --new --depth 1（增量模式）
✅ 每次获取评论后必须等待随机间隔（使用 execute_with_delay 模板）
```

### 📋 执行模板

```bash
# ═══ 首次全量获取 ═══
execute_with_delay 'node cli.js get <aweme_id> --all --depth 1' read

# ═══ 后续增量获取（推荐）═══
execute_with_delay 'node cli.js get <aweme_id> --new --depth 1' read
```

### 💡 原因

- `--all` 会获取所有评论，请求量大，容易触发抖音风控
- `--new` 只获取新评论，通常 1-2 次请求即可完成
- 如果需要更多评论，可以使用 `--pages N` 指定页数，但要谨慎使用

## 11. 错误处理

### 命令执行失败

```bash
# 在 execute_with_delay 函数中已处理
if [ $? -eq 0 ]; then
  echo "✓ 成功，等待 ${delay} 秒..."
  sleep "$delay"
else
  echo "✗ 失败，跳过等待"
  # 记录失败原因，继续执行下一条命令
fi
```

### 风控触发（status_code=8/2053）

```
1. 立即停止本轮执行
2. 记录触发风控的命令和错误码
3. 等待 30 分钟后再尝试
4. 在执行报告中详细记录
```

### 网络超时

```
1. 不要立即重试（可能是风控）
2. 等待 60 秒后重试一次
3. 如果仍然失败，停止本轮
4. 记录到执行报告
```

### 评论发布但看不到

```
1. 正常延迟（comment.status:7 审核中）
2. 等待 1-2 分钟再查，不是错误
3. 如果仍看不到，可能是被风控拦截
```
