# 评论区运营工作流

## 目标

提升互动率、活跃评论区、识别高价值用户。

## 前置条件（必须全部完成）

```
□ 已加载 用户配置.md(./用户配置.md)
□ 已加载 全局规则.md(./全局规则.md)（含评论筛选过滤规则）
□ Bridge Server 在线：bash ~/.claude/skills/douyin-cli/scripts/bridge.sh ensure
□ 油猴脚本已连接：node cli.js status
□ REPLIED_CIDS 集合已初始化（使用执行模板.md中的 init_replied_cids 函数）
```

## 🔴 核心安全规则（必须严格遵守）

### 规则 1：命令串行 + 强制随机间隔

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 严禁并发执行任何 CLI 命令                                  │
│  ❌ 严禁使用固定间隔（必须每次重新随机）                         │
│  ❌ 严禁跳过 sleep（即使"感觉"不需要）                          │
│                                                              │
│  ✅ 所有命令必须逐条串行执行                                   │
│  ✅ 写操作每条命令完成后必须等待 40-55 秒随机间隔                │
│  ✅ 读操作每条命令完成后必须等待 30-50 秒随机间隔                │
│  ✅ 每次等待前必须重新生成随机数                                │
└─────────────────────────────────────────────────────────────┘
```

**强制执行模板**（复制使用，不要修改结构）：

```bash
# ═══ 执行单条命令的模板 ═══
execute_with_delay() {
  local cmd="$1"
  local type="${2:-write}"  # write 或 read
  local delay
  
  if [ "$type" = "read" ]; then
    delay=$((30 + RANDOM % 21))  # 30-50 秒随机值
  else
    delay=$((40 + RANDOM % 16))  # 40-55 秒随机值
  fi
  
  echo "▶ 执行: $cmd"
  eval "$cmd"
  
  if [ $? -eq 0 ]; then
    echo "✓ 成功，等待 ${delay} 秒..."
    sleep "$delay"
  else
    echo "✗ 失败，跳过等待"
  fi
}
```

### 规则 2：不重复回复

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 同一条评论 cid 一生只能被回复一次（跨日跨轮均生效）           │
│                                                              │
│  ✅ 每次回复前必须检查 cid 是否在 REPLIED_CIDS 中              │
│  ✅ 回复成功后必须将 cid 加入 REPLIED_CIDS                    │
└─────────────────────────────────────────────────────────────┘
```

**强制执行流程**：

```bash
# ═══ 检查是否已回复 ═══
is_replied() {
  local cid="$1"
  grep -q "$cid" /tmp/replied_cids.txt 2>/dev/null
  return $?
}

# ═══ 回复前检查 ═══
if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
else
  execute_with_delay 'node cli.js post <aweme_id> "内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
fi
```

## 评论分类决策树

```
读取一条评论 (text, cid, uid):

  if cid in REPLIED_CIDS:                  # ⚠️ §2.1 硬性禁令
      skip "已回复过此评论"
      continue

  if text 是纯表情 / 贴纸 / 单字符:
      → 表情回复路线 (≤2个积极表情，无文字)
      可选: 🌹 / 🤝 / 😊 / 👍 / ❤️ / 🎉
      根据原评论情绪选择: 玫瑰花/握手/笑脸 → 友善; 红心/庆祝 → 兴奋

  elif text 长度 < 4 且无明确语义 ("666" / "哈哈" / "..."):
      skip "无意义水评"

  elif text 命中"优先回复"信号:
      ① 提问求助 (含 怎么/如何/为什么/?)
      ② 高质量反馈 (>15 字，提到视频具体细节)
      ③ 负面情绪 (抱怨/失望/愤怒)
      → 文字回复路线 (按 §3 偏好生成 15-80 字回复)

  else:
      skip "不匹配优先回复条件"
```

## 表情回复细则

- **触发条件**：评论文本去除空格后**只剩** emoji / 贴纸 unicode / 单字标点。
- **匹配映射**（建议，agent 可灵活选择）：
  - 用户发 ❤️ / 🌹 / 😘 → 回 🌹 或 ❤️
  - 用户发 👍 / 💪 / 🔥 → 回 🤝 或 💪
  - 用户发 😂 / 🤣 → 回 😄 或 😂
  - 用户发 🎉 / 🥳 → 回 🎉 或 ❤️
  - 无法判断 → 默认 🌹
- **不写文字**，纯表情即可（成本低、互动率高）。
- 表情回复同样占用配额，并且同样写入 `REPLIED_CIDS`。

## 执行流程

### Step 1: 获取我的作品

先获取自己已发布的作品，了解当前内容方向：

```bash
# ═══ 获取我的作品列表（使用执行模板）═══
execute_with_delay 'node cli.js my' read
```

**分析要点**：
- 记录作品的主要话题/领域（如 AI、编程、Agent 等）
- 对于标题信息不足的作品，获取详情补充了解

### Step 2: 获取作品评论

获取目标作品的评论：

```bash
# ═══ 获取作品评论（使用执行模板，优先使用增量模式）═══
execute_with_delay 'node cli.js get <aweme_id> --new --depth 1' read
```

**⚠️ 重要：增量获取评论优先使用 `--new`**
- ❌ 错误：每次全量获取所有评论（浪费请求，可能触发风控）
- ✅ 正确：`node cli.js get <aweme_id> --new --depth 1`（只获取新评论，安全高效）

**评论筛选**（遵循 `全局规则.md` 第 7 节）：
- 跳过：纯表情/贴纸、简短感叹、无意义水评
- 优先：提问求助、高质量反馈、负面情绪

### Step 3: 生成评论内容

1. 加载 `评论风格指南.md(./评论风格指南.md)`
2. 根据评论内容选择合适的风格（每轮至少用 3 种）
3. 生成回复内容（确保不违反 `全局规则.md` 的内容禁令）

**生成要点**：
- 有实质内容，能引发讨论
- 体现专业度或提供价值
- 符合抖音社区调性

### Step 4: 执行评论或回复

**发表评论**（对作品本身）：

```bash
# ═══ 对作品发表评论（使用执行模板）═══
execute_with_delay 'node cli.js post <aweme_id> "评论内容"'
```

**回复评论**（对其他用户的评论）：

```bash
# ═══ 回复特定评论（使用执行模板 + REPLIED_CIDS 检查）═══
if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
else
  execute_with_delay 'node cli.js post <aweme_id> "回复内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
fi
```

**执行要点**：
- 写操作每条命令间隔 40-55 秒随机值（使用执行模板自动处理）
- 执行前检查 cid 是否在 REPLIED_CIDS 中
- 发布后将 cid 加入 REPLIED_CIDS

### Step 5: 记录执行结果

```bash
# ═══ 查看操作日志（使用执行模板）═══
execute_with_delay 'node cli.js log --tail 20' read
```

## 专属限制

- 每作品每轮最多回复 **30** 条。
- 同一 uid 每轮最多回复 **1** 条（不同评论也算）。
- 连续 **5** 条评论均不匹配回复条件 → 停止本轮。
- ⚠️ §2.1 全局禁令同时生效（每条 cid 一生一次）。

## 增量执行

后续执行时使用增量模式：

```bash
# ═══ 只获取新评论（使用执行模板，优先使用增量模式）═══
execute_with_delay 'node cli.js get <aweme_id> --new --depth 1' read
```

**增量策略**：
- 评论去重依赖 REPLIED_CIDS
- 优先处理新评论

## 注意事项

1. **频率控制**：宁可少做，不可触发风控（使用执行模板自动处理）
2. **内容质量**：每条评论都要有实质内容
3. **风格变化**：每轮至少用 3 种不同风格
4. **去重检查**：严格执行 REPLIED_CIDS 检查
5. **评论筛选**：遵循 `全局规则.md` 第 7 节的筛选规则
6. **间隔执行**：所有命令必须使用 `execute_with_delay` 模板
7. **增量获取**：优先使用 `--new` 模式获取评论

## 执行检查清单

**每执行 5 条命令后，必须检查**：

```
□ 是否所有命令都是串行执行的？
□ 写操作每条命令之间是否等待了 40-55 秒？
□ 读操作每条命令之间是否等待了 30-50 秒？
□ 是否检查了 REPLIED_CIDS？
□ 是否使用了 ≥ 3 种不同风格？
□ 是否违反了内容禁令？
□ 增量获取评论是否使用了 --new 模式？
```

**如果发现违规**：立即停止本轮，记录原因到执行报告。
